# SEO運用フロー

## 週間サイクル

| 曜日 | 作業 |
|---|---|
| 月 | 新規記事 |
| 火 | 既存記事の改善 |
| 水 | 新規記事 |
| 木 | 案件条件・収益導線の点検 |
| 金 | 新規記事 |
| 土 | Search Console分析 |
| 日 | 保守・翌週準備 |

## 毎日の基本手順

1. 毎日10:00 JSTに届くSMSまたはGitHub Issue通知を開く
2. 指定された作業を行う
3. 結果をIssueへ記録する
4. 完了した項目へチェックを付ける
5. 次回、CodexへIssueのURLまたは結果を渡す

## 新規記事の完了条件

- 検索意図が1つに絞られている
- 公式情報または一次情報を参照している
- 広告の有無を明示している
- 関連記事への内部リンクがある
- sitemap.xmlへ追加している
- 公開後にURL検査を申請している

## 改善の判断順

1. 表示回数があるのにクリックされないページ
2. 平均掲載順位が11〜30位のページ
3. 広告リンクへの導線が弱いページ
4. 情報が古いページ
5. 内容が重複しているページ
